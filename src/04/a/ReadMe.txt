4-1
クラスメソッドの中でインスタンス変数numにアクセスしてしまっている。
getNumをインスタンスメソッドとして定義しないと呼び出すことはできない。
public int getNum();
とするべき。

4-2
二つのメソッドsetNum()は返り値こそ異なるものの全く同じシグネチャをもつ。
この実装ではオーバーロードとして成立せず、重複定義を起こしてしまう。

4-3
PIはfinalをつけて定義されているため定数である。
3.14で初期化されているため、Constant()メソッド内で3.145に変更することはできない。

4-4
A: 5
B: 10
C: 55.12
C: 52.0
A: 5
C: 260.0

getMult(5, 1);
ではintとintの変数２つを引数にもつため、一番上のメソッドが呼び出される
getMult(5, 1, 2);
ではint,int,intの変数３つを引数にもつため、真ん中のメソッドが呼び出される
getMult(5.2, 10.6);
ではdoubleの変数2つを変数にもつため、一番下のメソッドが呼び出される。
getMult(getMult(20, 2.6), getMult(5,1));
では、返り値が変数として取られている二つのメソッドが左から先に呼び出される
getMult(20, 2.6)
ではint的なものとdoubleが変数にあるため、20がdoubleとして認識され、三番目のメソッドが呼び出される
getMult(5,1)
ではintとintの変数２つを引数にもつため、一番上のメソッドが呼び出される
二つの返り値はdoubleとintであるため、getMult(52.0, 5)として三番目のメソッドと認識され、
doubleである260.0が返される。

4-5
Stringのローカル変数yは
if (x < 0) { 
    String  y = “Java”; 
}
のブロック中で定義されているが、else内では定義されていない。
したがって、yの未定義によりコンパイルエラーとなる。
